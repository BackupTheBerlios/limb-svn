Структура файлов и папок LIMB и проекта.

LIMB содержим следующие подкаталоги в корневом каталоге:

LIMB_DIR 
	|
	+ - core – здесь находятся все практически все php классы, которые использует LIMB (за 
исключением тестов).
	|
	+ - design – каталог с шаблонами, картинками, css-файлами и прочее, что касается 
внешнего вида.
	|
	+ - var – каталог, где хранятся компилированные шаблоны, логи, закешированные файлы и 
прочее.
	|
	+ - tests – каталог с тестами.

Обратите внимание, что структура каталогов в проектах и в самом LIMB_DIR должны совпадать. 
Это требуется для работы механизма разрешения имен (линк на соответствующую страницу). 

Теперь более подробно рассмотрим структуру каждого каталога более подробно.

LIMB_DIR/core
	|
       + - actions – здесь хранятся все классы для реализации действий, так называемые actions 
controllers. Мы еще не пришли к окончательному выводу, должны ли actions содержать бизнес-
логику или нет, где должна находиться бизнес-логика вообще – не хватает опыта.  
Действия для различных классов объектов хранятся в каталоге с соответствующим именем, 
например, действия create_article_action, edit_article_action and delete_article_action хранятся в 
папке с путем LIMB/core/actions/article. 
       |
       + - controllers – здесь хранятся классы контроллеров объектов сайта. Они обычно 
определяют функциональность объектов сайта.
       |
       + - datasource – здесь находятся вспомогательные классы для реализации различных 
выборок для шаблонов. Используются совместно с тегами <fetch:SUB_BRANCH> < DATASOURCE>. 
Активность наших шаблонов определяется через эти классы.
       |
       + - db_tables – классы для работы с отдельными таблицами базы данных. Эти классы 
реализуют такие операции, как получение списка, создание/редактирование/удаление строки 
таблицы.
       |
       + - lib – каталог с библиотечными классами (ну с маленькими оговорками).
       |
       + - locale – здесь расположены файлы со свойствами локалей.
       |
       + - model – классы модели. Классы объектов сайта, политики доступа, система аудита и 
сбора статистики, response классы. Некоторые классы лежат здесь весьма условно. 
       |
       + - settings – артефакт от WACT. Там находятся файлы настройки для шаблонного 
механизма.
       |
       + - strings – файлы с локализованными строками.
       |
       + - template – View, шаблонный механизм и теги.
       |
       + - tree – классы для работы с иерархическими структурами
