Добавление объекта в любую ветку сайта.

Для создания нового объекта можно использовать скрипт create_object.php. Этот скрипт позволяет создавать объекты любого типа в любой ветке сайта. Обратите внимание, что доступ к нему не закрыт, поэтому любой может воспользоваться им в своих целях. Поэтому не оставляйте этот скрипт на публичном сайте. Мы решим эту проблему в ближайшем релизе и внесем создание новых объектов в один из классов LIMB (скорее всего в admin_page). Для создания нового объекта необходимо ввести класс объекта, его идентификатор и заголовок и указать путь до родительского элемента. Вы можете указать «/» в качестве родительского документа для создания полностью новой ветки. Если объект создан, то выведется соответствующее сообщение над формой. Обратите внимание, что скрипт create_object.php не проверяет уникальность идентификаторов в одной ветке дерева. 

После создания нового объекта, если это был первый объект какого-либо класса, необходимо настроить права доступа к действиям для класса. Подробнее о правах доступа можно читать на странице с политикой доступа.

После установки прав доступа можно начать работать с объектом. Обычно, если это корневой объект для какой-либо отдельной фукнциональности (например, папка для новостей или же папка для каталога), то можно сделать пункт навигации для быстрого доступа к нему. Обратите внимание, что для отображения административного списка обычно используется действие admin_display для всех контентных объектов и просто display для системных объектов (папка с пользователями, папка с группами пользователей, навигация и т.д.). Возможно в будущем мы научим все объекты, которые должны отображаться в стиле центра администрирования, действию admin_display.

Необходимо кратко описать, что собой представляют шаблоны прав доступа.
