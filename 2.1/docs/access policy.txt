Политика доступа.

Политика доступа, используемая в LIMB является двух уровневой:
	1 уровень: права на объект – чтение и (или) запись(read, write)
	2 уровень: права на исполнение действий

Пользователь – объект класса пользователь. Хранится в общем дереве и подчиняется всем правилам контентных объектов. Пользователи могут входить в несколько групп одновременно.

Важно!!! Существует также специальный системный класс user, который отвечает за аутентификацию, сохранение данных в сессии и проч. Этот класс не следует путать с контентным классом пользователя. Контетный класс внутри использует системный класс, к примеру, код метода user_object::login :

	function login($login, $password, $locale_id = '')
	{
		$user =& user :: instance();
		return  $user->login($login, $password, $locale_id);
	}

Группа пользователей – объект класса группа пользователей. Также хранится в общем дереве и подчиняется всем правилам, применяемым к объектам.

В LIMB используются только разрешающие права, то есть пользователь имеет суммарный набор прав исходя из групп, в которые он входит + его собственные права.

 Права на объект. Пользователь или группа пользователей могут иметь 2 уровня доступа к объекту: право на чтение и право на запись.

Доступ к действиям. Действие – набор операций, который может быть выполнен над объектом. Для исполнения действия пользователь должен иметь право исполнять его. Права доступа к действиям задаются для классов объектов. Настройка прав доступа к действиям производится при помощи объекта класса class_folder (обычно он находится в базовой инсталляции по пути /root/admin/classes). Для каждого класса объектов указывается, к каким действиям какая группа пользователей имеет право доступа. В настоящее время задать права на действия можно только для групп пользователей. В нашей практике еще не встречалось случая, когда было бы необходимо устанавливать права доступа на действия для отдельных пользователей, однако мы планируем внести такую функциональность. Каждое действие требует определенного уровня доступа к объекту для своего исполнения (r или w или rw). Для того, чтобы пользователь мог исполнить действие над конкретным объектом, он должен иметь право на исполнение самого действия + необходимым уровнем доступа к объекту.
